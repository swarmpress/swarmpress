# Agent Scheduling Guide

> **Last Updated:** 2026-01-25

This guide explains how to configure and manage autonomous agent schedules in the swarm.press admin dashboard.

---

## Overview

Agent schedules allow your AI agents to perform maintenance tasks automatically without human intervention. Four schedule types are available:

| Schedule | Purpose | Default |
|----------|---------|---------|
| **Scheduled Content** | Publish content on scheduled dates | Daily 6 AM |
| **Media Check** | Validate images and media assets | Daily 7 AM |
| **Link Validation** | Find broken internal/external links | Weekly Monday 8 AM |
| **Stale Content** | Identify outdated content | Monthly 1st 9 AM |

---

## Accessing Schedules

1. Navigate to the **Admin Dashboard**
2. Select a website from the dropdown
3. Open the website detail page
4. Find the **Agent Schedules** section

## Creating Schedules

### Create Default Schedules

To create all four standard schedules at once:

1. Click **"Create Default Schedules"** button
2. All schedules are created with default timing
3. Schedules start in enabled state

### Create Individual Schedule

To create a specific schedule:

1. Click **"+ Add Schedule"**
2. Select schedule type from dropdown
3. (Optional) Customize cron expression
4. Click **"Create"**

### Pause a Schedule

To temporarily stop a schedule:

1. Find the schedule in the list
2. Click the **â¸ Pause** button

4. No executions will occur until resumed

### Resume a Schedule

To restart a paused schedule:

1. Find the paused schedule
2. Click the **â–¶ Resume** button
3. Schedule resumes from next scheduled time

## Schedule Types in Detail

### Scheduled Content

**What it does:**
- Finds content with `scheduled_for` date matching today
- Transitions content from "scheduled" to "published"
- Triggers site rebuild if needed

**When to adjust:**
- Change to earlier if you want content live before work hours
- Change to later for different timezone audience

### Media Check

**What it does:**
- Scans all pages for image references
- Validates each image URL is accessible
- Checks image dimensions and formats
- Reports broken or slow-loading images

**When to adjust:**
- Run more frequently for image-heavy sites
- Run after business hours to avoid load impact

### Link Validation

**What it does:**
- Crawls all internal links
- Checks external links (HTTP HEAD requests)
- Identifies broken links and orphan pages
- Creates tasks for fixes

**When to adjust:**
- Weekly is usually sufficient
- Run more often during major content changes

### Stale Content

**What it does:**
- Calculates freshness scores for all content
- Identifies content below threshold (default 70%)
- Creates editorial tasks for updates
- Notifies editors of high-priority items

**When to adjust:**
- Monthly works for most sites
- Run quarterly for stable content
- Run weekly for rapidly changing topics

## Troubleshooting

### Schedule Not Running

1. Check if schedule is enabled (not paused)
2. Verify Temporal server is running
3. Check cron expression is valid
4. Look for errors in execution history

### Execution Failing

1. Check error message in execution details
2. Verify agents have required permissions
3. Check external service connectivity
4. Review workflow logs in Temporal UI

### Missing Executions

1. Check if schedule was paused during that time
2. Verify Temporal was running
3. Check database for execution records
4. Review Temporal UI for schedule status

## Best Practices

### Timing Recommendations

- **Stagger schedules** - Don't run all at same time
- **Off-peak hours** - Run intensive tasks when traffic is low
- **Timezone awareness** - Consider your audience location

### Monitoring

- **Review history weekly** - Catch issues early
- **Set up alerts** - For consecutive failures
- **Track duration trends** - Identify performance issues

### Maintenance

- **Pause during updates** - Avoid conflicts with deployments
- **Test changes** - Use manual trigger before changing cron
- **Document custom timings** - Note why you changed defaults
